Strömchiffer

I dagens samhälle är vi vana vid att överföra känslig data över Internet. Exempelvis sköts en majoritet av våra bankärenden online. I medelanden som skickas mellan kund och bank finns information om personnummer, kontonummer och saldot på kontot. Dessa meddelande skickas ut på Internet dör de skickas från router till router tills de slutligen når fram till mottagaren. En egenskap som är viktig för tillförlitligheten är att vi inte från början vet vilka vägar våra meddelanden skickas. Detta innebär att det finns en risk någon kan snappa upp vårt meddelande och läsa dess innehåll. För att skydda våra meddelande från avlyssning kan ett strömchiffer eller annan typ av kryptering användas.

Den grundläggande funktionen för kryptering är att det meddelande som ska skyddas, även kallat klartext, omvandlas till ett krypterat meddelande. Det krypterade meddelandet kallas för kryptotext. Avsändare och mottagare har från innan sändning kommit överens om hur meddelandet ska krypteras. Detta innebär att sändaren kan omvandla klartexten till en kryptotext som mottagaren senare kan dekryptera och återskapa klartexten. För en avlyssnare som inte känner till alla detaljer om krypteringen ser kryptotexten ut som en massa symboler utan mönster eller betydelse.

Viktigt med publika metoder för kryptering.

Hemlighet kallas nyckel. Dessutom IV för att inte behöva nyckel så ofta. Kommer mer senare.

Samtidigt När man vill överföra information i en situation med risk för avlyssning kan ett strömchiffer användas.
För att kryptera data vid överföring eller lagring av data använder man symmetriska krypteringssyetem. I ett symmetriskt krypteringssystemen använder man samma krypteringsnyckel vid kryptering som vid dekryptering. En krypteringsnyckel består i moderna digitala system av en sekvens av de binära soffrorna, ett och noll. Det antal bitar som utgör nyckeln kallas för nyckellängd. För bra säkerhet kräver man i dagsläget minst 128-bitas nyckel. För extra känslig data rekommenderas 192 eller 256 bitar och för enklare system med begränsade resurser kan man nöja sig med 80-bitar. Man kan fråga sig varför man inte alltid använder många bitar i nyckeln. Anledningen är mycket enkel. Om vi har många bitar i nyckeln behöver vi mer minne eller mer CPU-kraft för att kryptera.

The courses folder may be renamed. For example, we can rename it to docs for software/project documentation or tutorials for creating an online course.

Delete tutorials

To remove these pages, delete the courses folder and see below to delete the associated menu link.

Update site menu

After renaming or deleting the courses folder, you may wish to update any [[main]] menu links to it by editing your menu configuration at config/_default/menus.toml.

For example, if you delete this folder, you can remove the following from your menu configuration:

[[main]]
  name = "Courses"
  url = "courses/"
  weight = 50

Or, if you are creating a software documentation site, you can rename the courses folder to docs and update the associated Courses menu configuration to:

[[main]]
  name = "Docs"
  url = "docs/"
  weight = 50

Update the docs menu

If you use the docs layout, note that the name of the menu in the front matter should be in the form [menu.X] where X is the folder name. Hence, if you rename the courses/example/ folder, you should also rename the menu definitions in the front matter of files within courses/example/ from [menu.example] to [menu.<NewFolderName>].
